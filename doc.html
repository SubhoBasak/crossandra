Crossandra v0.0.3 Documentation

Syntax

There is only one syntax in crossandra language. That's make it so easy to learn and remember. Here is the syntax:


<identifier>{
<state>: <property> <property>;
<state>: <property> <property>;
<state>: <property> <property>;
}

Identifier

Crossandra support only two kinds of identifier: ID based & Class based. ID based identifiers starts with a (#) symbol and class based identifiers starts with a (.) symbol. Identifiers name can only contain [a-zA-Z0-9], and the first letter must have to be [a-zA-Z]. Each identifier have to be unique.

State

Crossandra use 10 different types of states. States helps us to defind styles for small screen, large screen, dark mode etc.


  • @: This state is used to defind default styles.
  • sm: This state is used to define styles for small screen.
  • md: This state is used to defind styles for medium screen.
  • lg: This state is used to define styles for large screen.
  • xl: This state is used to defind styles for extra large screen.
  • hover: This state is used to defind styles for hover state.
  • focus: This state is used to define styles for focus state.
  • active: This state is used to define styles for active class.
  • deactive: This state is used to defind styles for deactive class.
  • dark: This state is used to define styles for dark mode.

Properties

Crossandra use few number of CSS properties to make the design more consistent and light weight. It's reduce the decision making time, because now you have to deal with few number of properties.


Dimension properties

  • width: There are total 20 width properties, starting from w-5 to w-100, each step is 5%. That's mean w-35, w-75, w-50 all are valid, but w-32, w-88 these are not valid.
  • min-width: Same as width property, but the porperty name has a prefix 'min-'. For example: min-w-50, min-w-75 etc.
  • max-width: Same as width property, but the porperty name has a prefix 'max-'. For example: min-w-50, min-w-75 etc.
  • height: Same as width property, but the porperty name starts with 'h-'. For example: h-50, h-75 etc.
  • min-height: Same as height property, but the porperty name has a prefix 'min-'. For example: min-h-50, min-h-75 etc.
  • max-height: Same as height property, but the porperty name has a prefix 'max-'. For example: max-h-50, max-h-75 etc.
  • overflow: This property is used to define overflow behaviour. It's variants are: "overflow-auto", "overflow-hide", "overflow-scroll", "overflow-visible".
  • overflow-x: This property is used to define overflow-x behaviour. It's variants are: "overflow-x-auto", "overflow-x-hide", "overflow-x-scroll", "overflow-x-visible".
  • overflow-y: This property is used to define overflow-y behaviour. It's variants are: "overflow-y-auto", "overflow-y-hide", "overflow-y-scroll", "overflow-y-visible".
  • object-fit: This property is used to define the object-fit behaviour. It's variants are: "fit-contain", "fit-cover", "fit-fill", "fit-scale"
